     111              :     AdaptBasedOnVelocity(δ_crit::Float64; safety_factor = 0.8)
     112              : 
     113              : Adapt timestep ``Δt`` based on the maximum velocity ``v_{\max}`` of filament nodes.
     114              : 
     115              : The objective is that the resulting maximum displacement ``δ_{\max} = v_{\max} Δt`` stays
     116              : below the given critical displacement `δ_crit`.
     117              : 
     118              : One application of this criterion is to ensure that reconnections happen in-between two
     119              : solver iterations (that is, to avoid that two filaments cross each other without
     120              : reconnecting). In this case, ``δ`` should be proportional to the chosen distance below
     121              : which reconnections are performed.
     122              : 
     123              : # Implementation details
     124              : 
     125              : This criterion is used in two ways:
     126              : 
     127              : 1. A priori, to decide the ``Δt`` to be used in the next iteration given the velocities of
     128              :    filament nodes at the start of the iteration (at time ``t``).
     129              :    This ensures ``Δt ≤ δ_{\text{crit}} / v_{\max}`` where ``v_{\max}`` is computed at the
     130              :    start of the iteration.
     131              : 
     132              : 2. A posteriori, after the actual node displacements ``δ`` from time ``t`` to time
     133              :    ``t + Δt`` have been computed (e.g. using some Runge--Kutta scheme).
     134              :    If some ``δ`` is larger than ``δ_{\text{crit}}``, then the iteration is recomputed after
     135              :    halving the timestep (``Δt → Δt/2``).
     136              :    In this case, the original displacements are thrown away (rejected).
     137              :    This process can be eventually repeated until the criterion is satisfied.
     138              : 
     139              : # Optional safety factor
     140              : 
     141              : The optional `safety_factor` should be `≤1`, which will further reduce the timestep
     142              : chosen a priori in step 1. This is to try to avoid too many rejected timesteps in step 2,
     143              : e.g. in case the actual advection velocity is larger than the velocity at the beginning of the
     144              : timestep.
     145              :
