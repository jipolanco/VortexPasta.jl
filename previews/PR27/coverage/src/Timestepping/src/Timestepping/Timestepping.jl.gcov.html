     382              : # Extended help
     383              : 
     384              : ## Difference between `callback` and `affect!`
     385              : 
     386              : The difference between the `callback(iter)` and `affect!(iter)` functions is the time at
     387              : which they are called:
     388              : 
     389              : - the `affect!` function is called *before* performing Biot-Savart computations from the
     390              :   latest filament positions. In other words, the fields in `iter.quantities` are not
     391              :   synchronised with `iter.fs`, and it generally makes no sense to access them.
     392              :   Things like energy estimates will be incorrect if done in `affect!`. On the other hand,
     393              :   the `affect!` function **allows to modify `iter.fs`** before Biot–Savart computations are
     394              :   performed.
     395              : 
     396              : - the `callback` function is called *after* performing Biot-Savart computations.
     397              :   This means that the fields in `iter.quantities` have been computed from the latest filament positions.
     398              :   However, one **must not modify `iter.fs`**, or otherwise the velocity `iter.vs` (which
     399              :   will be used at the next timestep) will no longer correspond to the filament positions.
     400              : 
     401              : ## Forcing / energy injection methods
     402              : 
     403              : ### Adding an external velocity
     404              : 
     405              : One can set the **`external_velocity`** keyword argument to impose an external velocity field
     406              : ``\bm{v}_{\text{f}}(\bm{x}, t)``.
     407              : In this case, the total velocity at a point ``\bm{x}`` (which can be on a vortex filament) will be
     408              : 
     409              : ```math
     410              : \bm{v}(\bm{x}) = \bm{v}_{\text{BS}}(\bm{x}) + \bm{v}_{\text{f}}(\bm{x})
     411              : ```
     412              : 
     413              : where ``\bm{v}_{\text{BS}}`` is the velocity obtained from the Biot–Savart law.
     414              : 
     415              : This can be a way of applying an external forcing (and injecting energy) to the vortex system.
     416              : 
     417              : The external velocity should be given as a function, which should have the signature `v_ext(x⃗::Vec3, t::Real) -> Vec3`.
     418              : The function should return a `Vec3` with the velocity at the location `x⃗` and time `t`.
     419              : For example, to impose a constant sinusoidal forcing in the ``x`` direction which varies along ``z``,
     420              : the `external_velocity` keyword argument could look like:
     421              : 
     422              :     external_velocity = (x⃗, t) -> Vec3(0.1 * sin(2 * x⃗.z), 0, 0)
     423              : 
     424              : One usually wants the external velocity to be divergence-free, to preserve the
     425              : incompressibility of the flow.
     426              : 
     427              : #### Notes on energy estimation
     428              : 
     429              : When setting an external velocity ``\bm{v}_{\text{f}}(\bm{x}, t)``, one may also want to
     430              : impose an external streamfunction field ``\bm{ψ}_{\text{f}}(\bm{x}, t)`` via the **`external_streamfunction`**
     431              : keyword argument.
     432              : Such a field plays no role in the dynamics, but it allows to include the kinetic energy
     433              : associated to the external velocity field when calling [`Diagnostics.kinetic_energy_from_streamfunction`](@ref).
     434              : If provided, the external streamfunction should satisfy ``\bm{v}_{\text{f}} = \bm{∇} × \bm{ψ}_{\text{f}}``.
     435              : 
     436              : More precisely, when applying an external velocity field, the total kinetic energy is
     437              : 
     438              : ```math
     439              : \begin{align*}
     440              :     E &= \frac{1}{2V} ∫ |\bm{v}_{\text{BS}} + \bm{v}_{\text{f}}|^2 \, \mathrm{d}^3\bm{x}
     441              :     \\
     442              :     &= \frac{1}{2V} \left[
     443              :     Γ ∮_{\mathcal{C}} \left( \bm{ψ}_{\text{BS}} + 2 \bm{ψ}_{\text{f}} \right) ⋅ \mathrm{d}\bm{s}
     444              :     +
     445              :     ∫ |\bm{v}_{\text{f}}|^2 \, \mathrm{d}^3\bm{x}
     446              :     \right],
     447              : \end{align*}
     448              : ```
     449              : 
     450              : where ``\bm{ψ}_{\text{BS}}`` is the streamfunction obtained from the Biot-Savart law.
     451              : Accordingly, when passing `external_streamfunction`, the streamfunction values attached to
     452              : vortex filaments (in `iter.ψs`) will actually contain
     453              : ``\bm{ψ}_{\text{BS}} + 2 \bm{ψ}_{\text{f}}`` (note the factor 2) to enable the
     454              : computation of the full kinetic energy.
     455              : Moreover, calling [`Diagnostics.kinetic_energy_from_streamfunction`](@ref) with a
     456              : [`VortexFilamentSolver`](@ref) will automatically include an estimation of the kinetic
     457              : energy of the external velocity field (the volume integral above).
     458              : 
     459              : !!! note
     460              : 
     461              :     Whether this total kinetic energy is of any physical interest is a different question,
     462              :     so it can be reasonable to ignore the `external_streamfunction` parameter in order to
     463              :     obtain the energy associated to ``\bm{v}_{\text{BS}}`` only.
     464              : 
     465              : ### Adding a stretching velocity
     466              : 
     467              : One can set the **`stretching_velocity`** keyword argument to impose a stretching velocity
     468              : ``\bm{v}_\text{L}(ξ, t)`` on filament locations ``\bm{s}(ξ, t)``.
     469              : This velocity will be parallel (and generally opposite) to the curvature vector
     470              : ``\bm{s}'' = ρ \bm{N}``, where ``ρ`` is the curvature and ``\bm{N}`` the unit normal.
     471              : 
     472              : More precisely, the `stretching_velocity` parameter allows to set a filament velocity of the
     473              : form
     474              : 
     475              : ```math
     476              : \bm{v}_{\text{L}}(ξ) = -v_{\text{L}}[ρ(ξ)] \, \bm{N}(ξ)
     477              : ```
     478              : 
     479              : where ``v_{\text{L}}`` is a velocity magnitude which can optionally depend on the local
     480              : curvature value ``ρ``.
