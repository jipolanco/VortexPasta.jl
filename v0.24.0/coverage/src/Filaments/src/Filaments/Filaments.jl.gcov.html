     450              : # Extended help
     451              : 
     452              : ## Evaluating high-order derivatives
     453              : 
     454              : By default the filament discretisation allows to evaluate up to the second derivative of the
     455              : curve with respect to its parametrisation.
     456              : This means in particular than one has access to the tangent, curvature and binormal vectors.
     457              : 
     458              : One may want to evaluate higher order derivatives, for example to also have access to the
     459              : curve torsion (which depends on the 3rd derivative).
     460              : This is only possible for some high-order discretisation methods ([`QuinticSplineMethod`](@ref),
     461              : as well as [`FourierMethod`](@ref) with the limitation that one has to evaluate on discretisation points).
     462              : Moreover, one needs to explicitly pass `nderivs = Val(3)` (or higher) to the `init` function.
     463              : 
     464              : ## Customising the filament parametrisation
     465              : 
     466              : By default the filament is parametrised as ``\bm{s}(t)`` such that the parameter ``t``
     467              : roughly corresponds to the arc length ``ξ``.[^1]
     468              : More precisely, the discrete values ``t_i`` at the discretisation points are defined from
     469              : the distances between discretisation points:
     470              : 
     471              : ```math
     472              : t_{i + 1} = t_{i} + |\bm{s}_{i + 1} - \bm{s}_i|, \quad t_1 = 0.
     473              : ```
     474              : 
     475              : One can use the `parametrisation` keyword argument to change this.
     476              : This may be a function `parametrisation(Xs, i)` which takes the vector `Xs` of
     477              : discretisation points and the index `i` associated to the start of the current segment.
     478              : 
     479              : For instance, to parametrise filaments according to the ``z`` increments between
     480              : discretisation points, one can pass:
     481              : 
     482              :     parametrisation = (Xs, i) -> abs(Xs[i + 1].z - Xs[i].z)
     483              : 
     484              : One can also pass a predefined parametrisation. See [`AbstractParametrisation`](@ref) for a list of options.
     485              :
