     383              : # Extended help
     384              : 
     385              : ## Difference between `callback` and `affect!`
     386              : 
     387              : The difference between the `callback(iter)` and `affect!(iter)` functions is the time at
     388              : which they are called:
     389              : 
     390              : - the `affect!` function is called *before* performing Biot-Savart computations from the
     391              :   latest filament positions. In other words, the fields in `iter.quantities` are not
     392              :   synchronised with `iter.fs`, and it generally makes no sense to access them.
     393              :   Things like energy estimates will be incorrect if done in `affect!`. On the other hand,
     394              :   the `affect!` function **allows to modify `iter.fs`** before Biot–Savart computations are
     395              :   performed. In particular, to inject new filaments, call [`inject_filament!`](@ref) from
     396              :   within the `affect!` function.
     397              : 
     398              : - the `callback` function is called *after* performing Biot-Savart computations.
     399              :   This means that the fields in `iter.quantities` have been computed from the latest filament positions.
     400              :   However, one **must not modify `iter.fs`**, or otherwise the velocity `iter.vs` (which
     401              :   will be used at the next timestep) will no longer correspond to the filament positions.
     402              : 
     403              : ## Forcing / energy injection methods
     404              : 
     405              : There are multiple implemented methods for affecting and injecting energy onto the vortex system.
     406              : These methods can represent different physical mechanisms (or in fact be quite unphysical), and some
     407              : of them can make more sense than others when simulating e.g. zero-temperature superfluids.
     408              : 
     409              : ### Adding an external velocity
     410              : 
     411              : One can set the **`external_velocity`** keyword argument to impose an external velocity field
     412              : ``\bm{v}_{\text{f}}(\bm{x}, t)``.
     413              : In this case, the total velocity at a point ``\bm{x}`` (which can be on a vortex filament) will be
     414              : 
     415              : ```math
     416              : \bm{v}(\bm{x}) = \bm{v}_{\text{BS}}(\bm{x}) + \bm{v}_{\text{f}}(\bm{x})
     417              : ```
     418              : 
     419              : where ``\bm{v}_{\text{BS}}`` is the velocity obtained from the Biot–Savart law.
     420              : 
     421              : This can be a way of applying an external forcing (and injecting energy) to the vortex system.
     422              : 
     423              : The external velocity should be given as a function, which should have the signature `v_ext(x⃗::Vec3, t::Real) -> Vec3`.
     424              : The function should return a `Vec3` with the velocity at the location `x⃗` and time `t`.
     425              : For example, to impose a constant sinusoidal forcing in the ``x`` direction which varies along ``z``,
     426              : the `external_velocity` keyword argument could look like:
     427              : 
     428              :     external_velocity = (x⃗, t) -> Vec3(0.1 * sin(2 * x⃗.z), 0, 0)
     429              : 
     430              : One usually wants the external velocity to be divergence-free, to preserve the
     431              : incompressibility of the flow.
     432              : 
     433              : #### Notes on energy estimation
     434              : 
     435              : When setting an external velocity ``\bm{v}_{\text{f}}(\bm{x}, t)``, one may also want to
     436              : impose an external streamfunction field ``\bm{ψ}_{\text{f}}(\bm{x}, t)`` via the **`external_streamfunction`**
     437              : keyword argument.
     438              : Such a field plays no role in the dynamics, but it allows to include the kinetic energy
     439              : associated to the external velocity field when calling [`Diagnostics.kinetic_energy_from_streamfunction`](@ref).
     440              : If provided, the external streamfunction should satisfy ``\bm{v}_{\text{f}} = \bm{∇} × \bm{ψ}_{\text{f}}``.
     441              : 
     442              : More precisely, when applying an external velocity field, the total kinetic energy is
     443              : 
     444              : ```math
     445              : \begin{align*}
     446              :     E &= \frac{1}{2V} ∫ |\bm{v}_{\text{BS}} + \bm{v}_{\text{f}}|^2 \, \mathrm{d}^3\bm{x}
     447              :     \\
     448              :     &= \frac{1}{2V} \left[
     449              :     Γ ∮_{\mathcal{C}} \left( \bm{ψ}_{\text{BS}} + 2 \bm{ψ}_{\text{f}} \right) ⋅ \mathrm{d}\bm{s}
     450              :     +
     451              :     ∫ |\bm{v}_{\text{f}}|^2 \, \mathrm{d}^3\bm{x}
     452              :     \right],
     453              : \end{align*}
     454              : ```
     455              : 
     456              : where ``\bm{ψ}_{\text{BS}}`` is the streamfunction obtained from the Biot-Savart law.
     457              : Accordingly, when passing `external_streamfunction`, the streamfunction values attached to
     458              : vortex filaments (in `iter.ψs`) will actually contain
     459              : ``\bm{ψ}_{\text{BS}} + 2 \bm{ψ}_{\text{f}}`` (note the factor 2) to enable the
     460              : computation of the full kinetic energy.
     461              : Moreover, calling [`Diagnostics.kinetic_energy_from_streamfunction`](@ref) with a
     462              : [`VortexFilamentSolver`](@ref) will automatically include an estimation of the kinetic
     463              : energy of the external velocity field (the volume integral above).
     464              : 
     465              : !!! note
     466              : 
     467              :     Whether this total kinetic energy is of any physical interest is a different question,
     468              :     so it can be reasonable to ignore the `external_streamfunction` parameter in order to
     469              :     obtain the energy associated to ``\bm{v}_{\text{BS}}`` only.
     470              : 
     471              : ### Adding a stretching velocity
     472              : 
     473              : One can set the **`stretching_velocity`** keyword argument to impose a stretching velocity
     474              : ``\bm{v}_\text{L}(ξ, t)`` on filament locations ``\bm{s}(ξ, t)``.
     475              : This velocity will be parallel (and generally opposite) to the curvature vector
     476              : ``\bm{s}'' = ρ \bm{N}``, where ``ρ`` is the curvature and ``\bm{N}`` the unit normal.
     477              : 
     478              : More precisely, the `stretching_velocity` parameter allows to set a filament velocity of the
     479              : form
     480              : 
     481              : ```math
     482              : \bm{v}_{\text{L}}(ξ) = -v_{\text{L}}[ρ(ξ)] \, \bm{N}(ξ)
     483              : ```
     484              : 
     485              : where ``v_{\text{L}}`` is a velocity magnitude which can optionally depend on the local
     486              : curvature value ``ρ``.
     487              : The `stretching_velocity` parameter should therefore be a function `v(ρ::Real) -> Real`.
     488              : 
     489              : Note that ``\bm{v} ⋅ \bm{s}''`` can be roughly interpreted as a local stretching rate
     490              : (this is actually true for the integrated quantity, see [`Diagnostics.stretching_rate`](@ref)).
     491              : Therefore, one may want ``v_{\text{L}}`` to be inversely proportional to the local
     492              : curvature ``ρ``, so that the local stretching rate is approximately independent of the
     493              : filament location ``ξ``.
     494              : To achieve this, one may set
     495              : 
     496              : ```julia
     497              : stretching_velocity = ρ -> min(γ / ρ, v_max)
     498              : ```
     499              : 
     500              : where `γ` is a constant (units ``T^{-1}``) setting the stretching magnitude, and `v_max` is
     501              : a maximum stretching velocity used to avoid very large velocities in low-curvature regions.
     502              : One could do something fancier and replace the `min` with a smooth regularisation such as
     503              : 
     504              : ```julia
     505              : stretching_velocity = ρ -> -expm1(-ρ / ρ₀) * (γ / ρ)  # note: expm1(x) = exp(x) - 1
     506              : ```
     507              : 
     508              : for some small curvature ``ρ₀``. The maximum allowed velocity will then be `vmax = γ / ρ₀`.
     509              : 
     510              : ### Forcing via a normal fluid
     511              : 
     512              : Alternatively to the above approaches, one may impose a "normal" fluid velocity field to
     513              : modify, via a mutual friction force, the velocity of the vortex filaments. To do this, one
     514              : first needs to construct a [`NormalFluidForcing`](@ref) (see that link for definitions and examples)
     515              : representing a normal fluid velocity field. Then, the obtained object should be passed as
     516              : the optional `forcing` argument.
     517              : 
     518              : In principle, this method can be combined with the previous ones. The normal fluid forcing
     519              : will be applied _after_ all the other forcing methods.
     520              : In other words, the vortex velocity ``\bm{v}_{\text{s}}`` used to compute the mutual
     521              : friction velocity includes all the other contributions (i.e. external and stretching velocities).
     522              : 
     523              : ### Injecting filaments over time
     524              : 
     525              : Another way of injecting energy is simply by adding vortices to the simulation from time to
