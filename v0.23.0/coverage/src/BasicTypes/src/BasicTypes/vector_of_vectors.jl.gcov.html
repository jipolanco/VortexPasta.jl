       5              :     VectorOfVectors(data::AbstractVector{<:AbstractVector})
       6              : 
       7              : Contains a list of vectors.
       8              : 
       9              : It behaves as much as possible as a basic vector of vectors. For instance:
      10              : 
      11              : - its `length` is the number of contained vectors;
      12              : 
      13              : - its element type (`eltype`) is the type of a contained vector;
      14              : 
      15              : - it only allows linear indexing (e.g. `u[i]` to get a single vector).
      16              :   To get an individual element (of type `T`), one should do `u[i][j]`.
      17              : 
      18              : Note that the individual vectors `u[i]` can have different lengths from each other.
      19              : 
      20              : There are also some differences:
      21              : 
      22              : - it overloads broadcasting mechanisms, so that doing `@. u = 2 * u + v` (where both
      23              :   variables are `VectorOfVectors`) works as expected and is efficient;
      24              : 
      25              : - copying a `VectorOfVectors` recursively copies its contained vectors, instead of just
      26              :   copying array references ("pointers").
